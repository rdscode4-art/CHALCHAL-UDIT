ISSUE
-----
"Ride is assigned this is notification api - check why the notification is 
not coming in the driver"

ROOT CAUSE
----------
API returns notification data with locations embedded in a "message" field,
but the driver app was looking for direct "pickup" and "destination" fields
that don't exist in the API response.

Result: Every notification was skipped (0% shown).

SOLUTION
--------
Added location parsing logic to extract coordinates from the message text.

Key Changes:
1. New method: _parseLocationsFromMessage() - Parses "from X to Y" pattern
2. Updated: _performPollingTick() - Extracts locations from message
3. Added: Status filtering - Only shows "pending" rides
4. Added: Fare parsing - Extracts "Fare: ₹500" from message
