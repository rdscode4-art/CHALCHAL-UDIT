                    ARCHITECTURE: BEFORE vs AFTER
================================================================================

BEFORE FIX (CRASHING):
─────────────────────────────────────────────────────────────────────────────

    main()
      ↓
    [Widgets.ensureInitialized()]
      ↓
    [System.orientation()]
      ↓
    Firebase.initializeApp()  ← FIRST INIT
      ↓
    FirebaseNotificationService.initializeNotifications()
      ↓
      └─→ Firebase.initializeApp()  ← SECOND INIT (DUPLICATE!)
            ↓
            ❌ NULL POINTER EXCEPTION
            ❌ Listener setup fails
            ↓
            FirebaseMessaging.onBackgroundMessage()  ❌ CRASHES HERE
            FirebaseMessaging.onMessage.listen()     (never reached)
            FirebaseMessaging.onMessageOpenedApp()   (never reached)
      ↓
    💥 APP CRASHES - USER SEES "App has stopped"


AFTER FIX (RESILIENT):
─────────────────────────────────────────────────────────────────────────────

    main()
      ↓
    TRY {
      ├─ [Widgets.ensureInitialized()]
      ├─ [System.orientation()]
      ├─ Firebase.initializeApp()  ✅ ONCE (correct)
      └─ FirebaseNotificationService.initializeNotifications()
           ├─ (removed duplicate init)
           ├─ TRY {
           │   └─ FirebaseMessaging.onBackgroundMessage()  ✅ OK or caught
           │ } CATCH { log warning }
           ├─ TRY {
           │   └─ FirebaseMessaging.onMessage.listen()    ✅ OK or caught
           │ } CATCH { log warning }
           └─ TRY {
               └─ FirebaseMessaging.onMessageOpenedApp()  ✅ OK or caught
             } CATCH { log warning }
    } CATCH {
      debugPrint("Firebase init error")
      debugPrint("App continues with limited features")
    }
      ↓
    runApp(RideGoApp)
      ↓
    ✅ APP LAUNCHES
      ├─ User can login
      ├─ User can book rides
      ├─ Notifications work (if Firebase OK)
      └─ Everything works (if Firebase fails, app still runs)

CRASH POINT COMPARISON:
─────────────────────────────────────────────────────────────────────────────

SCENARIO: User opens app with no internet

BEFORE:
  main() → Firebase.initializeApp() (fails)
        → No error handling
        → 💥 CRASH
        → "App has stopped"
        → User can't use app

AFTER:
  main() → try {
    Firebase.initializeApp() (fails)
  } catch {
    log error, continue
  }
  → runApp()
  → ✅ App starts
  → User can login and use app
  → Firebase features disabled
  → When internet returns, Firebase works

STATE TRANSITIONS:
─────────────────────────────────────────────────────────────────────────────

BEFORE (Linear, fragile):
  [init] → Firebase OK? → [Listeners OK?] → [Startup complete]
              ↓               ↓
              NO              NO
              ↓               ↓
           💥CRASH         💥CRASH
           Cannot proceed  Cannot proceed

AFTER (Parallel error handling):
  [init] → TRY Firebase
            CATCH → Continue with degraded mode
         ↓
  [Listeners] → TRY each separately
                CATCH each → Continue
         ↓
         [Startup complete]
         (with features working or disabled)
