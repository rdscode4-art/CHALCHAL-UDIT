APP WAS CRASHING AUTOMATICALLY? FIXED!

THREE CRITICAL ISSUES IDENTIFIED & RESOLVED:

1. FIREBASE DOUBLE INITIALIZATION ✅ FIXED
   Location: lib/core/services/firebase_notification_service.dart
   Issue: Firebase.initializeApp() called twice (main.dart + service)
   Impact: Null pointer exception, listener setup failed
   Fix: Removed duplicate initialization
   
2. UNHANDLED FIREBASE LISTENER FAILURES ✅ FIXED
   Location: lib/core/services/firebase_notification_service.dart
   Issue: No error handling on Firebase message listeners
   Impact: Any listener setup failure = app crash
   Fix: Added try-catch around 3 listeners:
        - onBackgroundMessage
        - onMessage  
        - onMessageOpenedApp
   
3. NO APP INIT ERROR HANDLING ✅ FIXED
   Location: lib/main.dart
   Issue: Firebase init failures not caught
   Impact: Single Firebase failure = app never starts
   Fix: Added try-catch in main(), graceful degradation
